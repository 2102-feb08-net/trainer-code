Images

Now let's include an image - we'll use the <img> tag.

The image tag above contains two attributes - src and alt. The value of the "src" attribute specifies the URL of the image, and the "alt" attribute allows you to display alternative text if the image cannot be loaded for some reason.

The image tag is a self-closing tag, meaning that instead of having a separate tag to end the element, the tag itself ends the element with a /> and no other tags are nested within it.

We can also set the height and width of an image, using the height and width attributes. Below, we made the cat image 75 × 50 pixels.

Tables

Tables can be included in HTML using the - yes, you guessed it - <table> tag. There are actually several different table tags, listed below. Optional tags are marked with an *.

  • *<caption> - defines a caption for the table. If used, must go directly after the <table> tag
  • *<thead> - defines the header row for the table
  • *<th> - defines a table header cell
  • *<tbody> - defines the body of the table. Not needed if header not specified
  • *<tfoot> - groups the footer content of the table
  • <tr> - defines a table row
  • <td> - defines a table data cell in the body

Forms

Forms are arguably one of the most important types of tags - you'll use forms to gather input from the user. A list of common form tags are listed below:

  • <form> - for specifying an entire form
  • <input> - some kind of user input, depending on the type attribute:
    • text - for inline text
    • number
    • color - displays a color picker
    • date - displays a date picker
    • file
    • range - specify a range of numbers with min and max attributes
    • tel - telephone number
    • password - will hide the characters entered
    • radio - a radio button (group of options where the user can only select one)
    • checkbox - user can select multiple options
  • <button> - a user-clickable button
  • <select> - a dropdown menu with options specified by the option tag
  • <textarea> - a multi-lined, resizable text field
  • <label> - defines a label for an input element

Form inputs can have the name attribute to identify the fields. The values of the name attribute are used as the keys for the user data when the form is submitted and sent to a server for processing.

HTML5 Semantic Elements

HTML5 is the newest version of HTML, and many existing tags were deprecated while new tags were introduced. The point of new HTML5 tags is to make HTML markup more semantic - meaning that the tags should reflect the meaning of the content, defining more clearly what the content is, not simply how it is displayed. Here are some HTML5 semantic tags:

  • <header> - defines heading of a page
  • <footer> - defines footer of a page
  • <article> - defines an article
  • <aside> - content aside from the main content
  • <nav> - navigation section
  • <main> - the main content of the page
  • <section> - logically split content into sections
  • <details> - additional details the user can view or hide
  • <summary> - visible heading for "details" element
  • <mark> - marked / highlighted text

The most useful of these will be the header, footer, and nav tags. Instead of using divs for each of these sections, the semantic tags can be used.
